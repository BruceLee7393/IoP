# 订单批量导入功能

## 1. 功能概述

订单批量导入功能允许用户通过上传Excel文件批量导入订单数据，系统支持自动解析层次结构（订单→组件→子组件→软件），并提供错误信息提示。

---

## 2. 相关文件

```
OrderView.vue
    ├─ useOrderManagement.js (配置)
    ├─ OrderImportDialog.vue (对话框)
    │   ├─ importConfig.js (可选，若使用通用组件)
    │   └─ order.js (API)
    │       ├─ http.js (HTTP封装)
    │       └─ apiUtils.js (工具)
    └─ CrudTable.vue (列表组件)

xlsxUtils.js (独立工具，按需引入)
fileUtils.js (独立工具，按需引入)
```

###  数据流向

```
用户操作 → 选择Excel文件 → 前端FormData封装 → HTTP请求
                                              ↓
                                         后端解析验证
                                              ↓
                          ┌─────────────────┴─────────────────┐
                          ↓                                    ↓
                     全部成功                            存在错误
                (返回JSON)                         (返回Excel文件)
                          ↓                                    ↓
              显示成功提示                          下载错误文件
              刷新订单列表                        用户修改后重新导入
```

### 3.3 order.js (API层)

**路径**: `src/api/order.js`

- 封装订单相关的所有API请求
- 处理导入相关的特殊响应类型（JSON/Excel）
- 提供统一的错误处理

**核心API - 上传导入文件**:

```javascript
export const uploadOrderImportFile = async (file) => {
  const formData = new FormData()
  formData.append('file', file)

  try {
    const response = await request.post('/orders/import', formData, {
      responseType: 'blob'  // 关键：使用blob接收，支持多种响应类型
    })

    const contentType = response.headers['content-type']
    
    // 场景1: 全部导入成功（返回JSON）
    if (contentType.includes('application/json')) {
      const text = await response.data.text()
      const result = JSON.parse(text)
      return {
        success: true,
        type: 'SUCCESS',
        data: result,
        success_count: result.success_count || 0,
        error_count: 0
      }
    }
    
    // 场景2: 存在错误（返回Excel文件）
    if (contentType.includes('spreadsheet')) {
      const blob = new Blob([response.data])
      
      // 从响应头获取统计信息
      const success_count = parseInt(response.headers['x-success-count']) || 0
      const error_count = parseInt(response.headers['x-error-count']) || 0
      
      // 判断是部分成功还是全部失败
      const isPartialSuccess = success_count > 0
      
      return {
        success: isPartialSuccess,
        type: isPartialSuccess ? 'PARTIAL_SUCCESS' : 'ERROR',
        hasErrorFile: true,
        blob: blob,
        filename: extractFilename(response),
        success_count,
        error_count,
        message: isPartialSuccess 
          ? `部分导入成功：${success_count}条成功，${error_count}条失败` 
          : '导入失败，请下载错误文件查看详情'
      }
    }
    
  } catch (error) {
    // 错误处理逻辑
    handleImportError(error)
  }
}
```

1. **responseType: 'blob'**
   - 原因：后端可能返回JSON或Excel文件，使用blob可同时接收两种类型
   - 好处：避免二进制数据被错误解析

2. **Content-Type判断**
   - JSON (`application/json`): 全部成功
   - Excel (`application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`): 有错误

3. **自定义响应头**
   ```
   X-Success-Count: 10    // 成功导入数量
   X-Error-Count: 5       // 失败数量
   X-Total-Count: 15      // 总数量
   ```

4. **文件名提取**
   ```javascript
   const contentDisposition = response.headers['content-disposition']
   const filename = contentDisposition.match(/filename="?([^"]+)"?/)[1]
   ```

**下载模板API**:

```javascript
export const downloadOrderImportTemplate = async () => {
  const response = await request.get('/orders/import/template', {}, {
    responseType: 'blob'
  })
  
  // 创建下载链接
  const blob = new Blob([response.data], {
    type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet'
  })
  const url = window.URL.createObjectURL(blob)
  const link = document.createElement('a')
  link.href = url
  link.download = extractFilename(response)
  link.click()
  window.URL.revokeObjectURL(url)
}
```

### 3.4 xlsxUtils.js (Excel处理工具)

**路径**: `src/utils/xlsxUtils.js`

- 按需加载xlsx库（减少初始包体积）
- 提供Excel读取、生成、下载功能
- 统一Excel处理逻辑

**核心功能**:

```javascript
// 1. 按需加载xlsx库
export const loadXlsx = async () => {
  if (!xlsx) {
    xlsx = await import('xlsx')
  }
  return xlsx
}

// 2. 读取Excel文件
export const readExcelFile = async (file, options = {}) => {
  const XLSX = await loadXlsx()
  
  return new Promise((resolve, reject) => {
    const reader = new FileReader()
    
    reader.onload = (e) => {
      const data = new Uint8Array(e.target.result)
      const workbook = XLSX.read(data, { type: 'array' })
      const sheetName = workbook.SheetNames[0]
      const worksheet = workbook.Sheets[sheetName]
      const jsonData = XLSX.utils.sheet_to_json(worksheet, options)
      resolve(jsonData)
    }
    
    reader.readAsArrayBuffer(file)
  })
}

// 3. 生成Excel文件
export const generateExcelFile = async (data, sheetName, options) => {
  const XLSX = await loadXlsx()
  
  const worksheet = XLSX.utils.json_to_sheet(data, options)
  const workbook = XLSX.utils.book_new()
  XLSX.utils.book_append_sheet(workbook, worksheet, sheetName)
  
  const excelBuffer = XLSX.write(workbook, {
    bookType: 'xlsx',
    type: 'array'
  })
  
  return new Blob([excelBuffer], {
    type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet'
  })
}

// 4. 下载Excel文件
export const downloadExcelFile = async (data, filename, options) => {
  const blob = await generateExcelFile(data, 'Sheet1', options)
  const url = window.URL.createObjectURL(blob)
  const link = document.createElement('a')
  link.href = url
  link.download = filename
  link.click()
  window.URL.revokeObjectURL(url)
}
```

- **按需加载**：xlsx库较大（~1MB），按需加载减少首屏加载时间
- **Promise封装**：将FileReader的回调转换为Promise，便于async/await使用
- **统一接口**：所有Excel操作通过统一工具函数，便于维护和测试

## 4. 交互流程

### 4.1 导入流程

```
1. 用户点击"批量导入"按钮
   ↓
2. 打开OrderImportDialog对话框
   ↓
3. 用户选择Excel文件
   ↓
4. 前端验证文件类型和大小
   ↓
5. 用户点击"开始导入"
   ↓
6. 构造FormData并上传
   ↓
7. 后端解析Excel并验证数据
   ↓
8. 判断响应类型
   ├─ JSON响应 (全部成功)
   │  ├─ 显示成功提示
   │  ├─ 显示成功数量
   │  └─ 2秒后自动关闭对话框并刷新列表
   │
   └─ Excel响应 (存在错误)
      ├─ 判断是否部分成功
      ├─ 自动下载错误文件
      ├─ 显示成功/失败统计
      └─ 用户关闭对话框
```

### 4.2 三种响应场景

#### 场景A: 全部导入成功

```
请求: POST /api/orders/import
响应:
  Status: 200
  Content-Type: application/json
  Body:
  {
    "success": true,
    "message": "订单导入成功",
    "success_count": 10,
    "imported_orders": [...]
  }

前端处理:
  1. 解析JSON
  2. 显示成功消息
  3. 2秒后关闭对话框
  4. 刷新订单列表
```

#### 场景B: 部分导入成功

```
请求: POST /api/orders/import
响应:
  Status: 200
  Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
  Headers:
    X-Success-Count: 7
    X-Error-Count: 3
    X-Total-Count: 10
    Content-Disposition: attachment; filename="order_import_errors_20260108123456.xlsx"
  Body: [Excel文件二进制数据]

前端处理:
  1. 检测Content-Type为Excel
  2. 读取X-Success-Count > 0，判定为部分成功
  3. 自动下载错误文件
  4. 显示"部分导入成功：7条成功，3条失败"
  5. 提示用户修改错误数据后重新导入
  6. 3秒后关闭对话框
  7. 刷新订单列表（显示成功导入的7条）
```

#### 场景C: 全部导入失败

```
请求: POST /api/orders/import
响应:
  Status: 200 (或 400)
  Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
  Headers:
    X-Success-Count: 0
    X-Error-Count: 10
    X-Total-Count: 10
    Content-Disposition: attachment; filename="order_import_errors_20260108123456.xlsx"
  Body: [Excel文件二进制数据，包含错误信息列]

前端处理:
  1. 检测Content-Type为Excel
  2. 读取X-Success-Count = 0，判定为全部失败
  3. 自动下载错误文件
  4. 显示"本次导入失败，返回文件已标注失败原因"
  5. 用户手动关闭对话框
  6. 不刷新列表（因为没有成功导入的数据）
```

### 4.3 错误Excel文件格式

导入失败时，后端返回的Excel文件会在原有数据基础上增加"错误信息"列：

用户可以：
1. 根据"错误信息"列修正数据
2. 删除已成功导入的行
3. 重新上传Excel文件

## 5. API接口

### 5.1 导入订单

**接口**: `POST /api/orders/import`

**请求**:
```http
POST /api/orders/import HTTP/1.1
Authorization: Bearer <token>
Content-Type: multipart/form-data

```
